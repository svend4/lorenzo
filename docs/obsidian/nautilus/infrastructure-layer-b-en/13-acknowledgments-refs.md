## Acknowledgments

This document emerged from the author's observation, near 
the end of an extended seven-document session, that the chat 
medium was insufficient for the kind of sustained 
intellectual project the documents represented. The 
observation crystallized a problem that had been implicit 
throughout the session but not articulated.

The position of this document in the series — designated 2.3 
rather than 8 — reflects the author's judgment that this 
inquiry document is structurally closer to early architectural 
work (Documents 1-2) than to applied work (Documents 3-7). 
It addresses infrastructure rather than application. The 
fractional numbering preserves the linear sequence of applied 
papers while inserting the infrastructure question where 
it logically belongs.

The author retained editorial authority. The framing, the 
recursive Nautilus possibility, and the practical 
recommendations all originated with the author. Claude 
contributed structure and reference work.

## References

The references for this document are sparser than for the 
applied papers, because the document is exploratory rather 
than specifying. Key relevant work includes:

### Threading and Annotation

- Hypothesis project (https://web.hypothes.is/) — open 
source web annotation
- Discourse forum software (https://www.discourse.org/) — 
modern threaded forum
- Federated Wiki experiments by Ward Cunningham — alternative 
models for collaborative writing

### Knowledge Management

- Sönke Ahrens (2017). *How to Take Smart Notes*. Cambridge 
University Press. (Zettelkasten method, foundational for 
modern personal knowledge management.)
- Andy Matuschak (ongoing). Working notes on tools for thought.

### AI Collaboration Infrastructure

- Anthropic Projects documentation (current)
- Microsoft Copilot Studio (different approach to similar 
problem)
- Various multi-agent frameworks (LangChain, CrewAI, AutoGen)

### Earlier Companion Documents in This Series

Documents 1-7 of the Nautilus / OKWF series, as listed in the 
header of this document.
