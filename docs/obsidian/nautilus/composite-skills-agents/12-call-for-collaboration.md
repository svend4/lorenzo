## 12. Call for Collaboration

This paper opens questions that require multi-stakeholder 
engagement.

### 12.1. For Practitioners

Tell us how your practice combines specializations. What 
configuration of sub-agents would actually serve your work? 
What would make composite agents valuable versus merely complex?

Pilot participation invited, particularly for the SGB Advocate 
Colleague pilot.

### 12.2. For Curators and Domain Experts

Sub-agent registry development needs curated knowledge across 
many narrow specializations. Domain experts willing to help 
develop sub-agents in their specialties — particularly 
compensated through foundation funding — would substantially 
accelerate deployment.

### 12.3. For Software Engineers and Architects

Coordinator infrastructure is substantial engineering work. 
Open-source contributions to reference implementations welcome. 
Particular needs:
- Sub-agent registry standards and protocols
- Configuration management interfaces
- Disagreement detection and presentation systems
- Audit and observability tools

### 12.4. For Researchers

Empirical studies of composite configuration patterns would 
illuminate questions raised in Section 11. Particular value in:
- Configuration clustering analysis
- Effectiveness comparison: composite vs. monolithic agents
- Long-term effects on professional development
- Cross-cultural deployment outcomes

### 12.5. For Foundation Funders

Composite Skills Agents represent higher-leverage infrastructure 
investment than monolithic Professional Colleague Agents:
- More flexible support across diverse practitioner needs
- More scalable across adjacent professions
- More aligned with how mastery actually develops
- More resilient to changes in any single component

Funding sub-agent development is investing in foundational 
infrastructure that supports many configurations across many 
practitioners.

### 12.6. For Policymakers

Regulatory frameworks for AI in professional practice will 
need to address composite architectures specifically. Questions 
about liability, professional responsibility, and quality 
standards apply differently when many specialized sub-agents 
contribute to outputs.

Engagement during regulatory development helps ensure frameworks 
match actual technology architecture.
