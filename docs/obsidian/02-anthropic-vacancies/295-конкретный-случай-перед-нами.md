## Конкретный случай перед нами

Семь документов, произведённых в этой сессии,
имеют конкретные инфраструктурные потребности,
которые текущие инструменты частично адресуют.

### Что у нас есть

Чат-сессия содержит документы и диалог,
который их произвёл. Через инструменты
conversation_search и recent_chats Claude
будущие сессии могут извлекать контекст из
этой. Это частичное сохранение.

### Что нам нужно, но отсутствует

**Тред из конкретных сообщений.** Автор может
захотеть вернуться к обсуждению рисков
Композитного Агента Навыков (раздел 10
Документа 7) и развить один конкретный риск
дальше. Текущий чат не имеет механизма для
этого. Тред должен будет начать новый чат с
цитированным контекстом.

**Аннотация на документах.** Когда автор
показывает документы потенциальным сотрудникам,
эти сотрудники захотят прокомментировать
конкретные разделы. В настоящее время это
требует, чтобы они либо посылали сообщения с
цитированным текстом, либо редактировали
документы напрямую. Нет среднего пути.

**Поддержание перекрёстных ссылок.** Когда
Документ 7 говорит «см. Документ 6 Раздел 7.5
для оригинального предложения SGB пилота», это
текст. Если Документ 6 переструктурирован
(разделы перенумерованы), ссылка ломается
бесшумно. Никакой механизм не обнаруживает это.

**Ветвящееся исследование.** Автор может
захотеть попробовать два разных подхода к
развёртыванию Композитного Агента Навыков,
развить оба, сравнить их и выбрать. В
настоящее время это потребовало бы ручного
поддержания двух версий.

**Продвижение в репозиторий.** Семь документов
должны быть закоммичены на GitHub. Это
прямолинейно — копировать, коммитить, пушить.
Но обсуждение, которое их произвело,
рассмотренные альтернативы, цепочки рассуждений
— всё это живёт только в чате. Репозиторий
будет иметь артефакты без происхождения.

**Селективный контекст для дальнейшей работы.**
Будущие сессии Claude о конкретных аспектах
этой работы нуждаются в конкретном контексте —
не во всех 50 000 словах. В настоящее время
обеспечение контекста означает либо загрузку
документов (теряется контекст чата), либо
использование conversation_search (ограничено).

### Что мы можем делать прагматично

Учитывая пробел, что мы можем реалистично
делать сейчас?

**Вариант 1 — Закоммитить и уйти.** Сохранить
документы на GitHub. Рассматривать их как
статические референсные артефакты. Будущая
разработка происходит в свежих сессиях с
документами как контекстом. Ограничения:
перекрёстные ссылки ломаются, нет аннотации,
нет тредов, но нет накладных расходов на
поддержку.

**Вариант 2 — Закоммитить плюс GitHub
Discussions.** Сохранить документы на GitHub.
Открыть GitHub Discussions для репозитория.
Использовать Discussions для тредированных
разговоров о конкретных аспектах. Ограничения:
культурное несоответствие, AI-сотрудничество
внешнее по отношению к Discussions, но тред
работает.

**Вариант 3 — Гибрид с Projects.** Сохранить
документы на GitHub. Создать Anthropic Project
для работы Nautilus / OKWF. Использовать
Project для AI-вспомогательной разработки
новых аспектов. Использовать GitHub для
стабильных артефактов. Использовать Discussions
для общественного ввода. Ограничения:
фрагментация, но каждый инструмент используется
для своих сильных сторон.

**Вариант 4 — Кастомная лёгкая установка.**
Для технических пользователей, готовых к
самохостингу, развернуть Discourse или
аналогичный тредированный форум. Соединить с
GitHub через интеграции. Использовать Anthropic
API для AI-сотрудничества через кастомный
интерфейс. Ограничения: бремя поддержки, но
больше согласовано с реальным рабочим
процессом.

Для большинства случаев Вариант 3 (гибрид)
наиболее реалистичен. Вариант 4 достижим для
конкретного случая, потому что автор имеет
техническую способность и существующую
инфраструктуру репозитория.
