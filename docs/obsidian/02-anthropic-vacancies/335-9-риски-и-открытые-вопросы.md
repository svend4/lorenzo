## 9. Риски и Открытые Вопросы

Несколько неопределённостей заслуживают явного
внимания.

### 9.1. Будущее Направление Cowork

Cowork — исследовательский предпросмотр.
Продуктовая дорожная карта Anthropic не
публично заявлена. Возможные сценарии:

**Сценарий 1 — Cowork созревает как
ожидается**: Добавляет командное разделение,
зрелую MCP интеграцию, шаблоны Project.
Позиционирование InGit прочное.

**Сценарий 2 — Cowork разворачивается**:
Anthropic перенаправляет к другой модели.
Некоторые текущие способности устаревают.
Подложка InGit по-прежнему полезна
(offline-first), но интеграция с Cowork слабее.

**Сценарий 3 — Cowork захвачен конкуренцией**:
OpenAI, Google запускают конкурирующие
продукты. Cowork остаётся способным, но
теряет долю рынка. InGit нужна более широкая
совместимость.

Смягчение: Ценность InGit за пределами Cowork
(offline-first, переносимый, структурированный)
обеспечивает жизнеспособность даже в
неблагоприятных сценариях. Не переусердствуйте
с Cowork-специфичными функциями, которые
теряют ценность при изменении Cowork.

### 9.2. Эволюция Стандарта MCP

MCP недавний (конец 2024). Стандарты всё ещё
эволюционируют. Риск, что значительные
изменения сломают существующие серверы.

Смягчение: MCP — открытый стандарт с
множественными реализаторами. Даже если
Anthropic конкретно изменит направление, MCP
продолжается. Стройте для MCP, не для
Cowork-специфичного протокола.

### 9.3. Потребление Ресурсов Cowork

Согласно опубликованным отчётам, сессии
Cowork потребляют значительно больше
выделения использования, чем чат (одна сессия
Cowork Project = 20+ обычных чатов). Для Pro
пользователей пределы могут быть достигнуты
быстро.

Смягчение: InGit Projects с Cowork должны быть
спроектированы для **эффективного
использования агента**, не для болтливого
взаимодействия. Структурированные выходы,
пакетные операции, запланированные задачи, а
не чат в реальном времени.

### 9.4. Конфиденциальность и Чувствительная Работа

Данные Cowork текут через инфраструктуру
Anthropic. Для очень чувствительной работы
(юридической, медицинской, финансовой) это
может быть неприемлемо.

Смягчение: Оффлайн режим InGit остаётся
ценным. Для чувствительной работы используйте
InGit в одиночку с ручными рабочими
процессами или локальные модели. Интеграция
Cowork для нечувствительных компонентов.

### 9.5. Способности Автора

Автор этой статьи имеет множественные активные
обязательства: SGB судебные дела, физические
соображения здоровья (GdB 70, Pflegegrad),
разработка Nautilus, восемь существующих
документов в этой серии.

Добавление активной работы по интеграции
InGit/Cowork может превысить реалистичные
способности.

Смягчение: Последовательный прогресс. Начать
с Пути 1 (без разработки). Расширять только
по мере того, как пропускная способность
позволяет. Не обязываться к амбициозным
временным рамкам.

### 9.6. Открытые Вопросы

**В1**: Откроет ли Anthropic шаблоны Project
для третьих сторон скоро? Неизвестно.

**В2**: Улучшится ли распространение MCP
серверов (реестр, управление пакетами)? В
настоящее время фрагментировано.

**В3**: Как эволюционирует ценообразование?
Cowork в настоящее время дорог в использовании;
если затраты упадут, принятие ускорится.

**В4**: Добавит ли Cowork функции тред/
ветвление/аннотация? Если да, заполняет больше
пробела Слоя B; если нет, оставляет
пространство для других.

**В5**: Появятся ли конкуренты? OpenAI, Google,
вероятно, построят аналогичные способности.
Стандартизация неясна.

Эти неопределённости формируют, но не
блокируют прогресс. Путь 1 работает независимо
от любого из них.
