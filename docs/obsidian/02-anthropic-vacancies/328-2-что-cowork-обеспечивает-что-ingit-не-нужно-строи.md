## 2. Что Cowork обеспечивает, что InGit не нужно строить

Если InGit позиционируется дополнять Cowork, а
не заменять его, несколько существенных функций
могут быть удалены из дорожной карты InGit.
Они теперь **обеспечиваются Cowork**, и InGit
не должен их дублировать.

### 2.1. Агентское выполнение задач

Ключевая способность Cowork: агентское
многошаговое выполнение задач. Claude в Cowork
может:
- Читать множественные файлы, синтезировать
  информацию
- Принимать решения о подходе
- Выполнять команды оболочки в изолированной
  VM
- Итерировать на основе результатов
- Координировать sub-агентов для параллельной
  работы
- Самопроверять выходы перед отчётом

InGit и не планировал строить это в любом
случае, но Cowork обеспечивает агентский слой,
который делает структурированное рабочее
пространство гораздо более ценным.

### 2.2. Постоянная Память Между Сессиями

Cowork Projects имеют ограниченную память.
Claude помнит в рамках Project между сессиями.
Чувствительные данные исключены. Пользователь
может видеть, редактировать, удалять
запомненное содержание.

Это что-то, что InGit мог бы хотеть в конечном
итоге. Не нужно сейчас.

### 2.3. UI для Работы со Знаниями

Cowork обеспечивает пользовательский интерфейс
для взаимодействия с Project. Файлы видны.
Задачи видны. Память видна. Подсказки на
разрешения для чувствительных операций.

Оригинальная дорожная карта InGit включала
Desktop GUI с Electron + React. С Cowork как
основным интерфейсом, **это становится
ненужным**. Сохраняет существенные усилия
разработки.

### 2.4. Экосистема Коннекторов

Cowork интегрируется с Gmail, Drive, Slack,
GitHub, Notion и многими другими. Через MCP
могут быть добавлены пользовательские
коннекторы.

InGit не планировал строить коннекторы сам, но
существующая экосистема Cowork означает, что
InGit Project может свободно использовать
внешние сервисы.

### 2.5. Способность Использования Компьютера

Когда коннекторы недоступны, Cowork может
навигировать по рабочему столу и браузеру
через использование компьютера. Это
исследовательский предпросмотр, но
функциональный.

Для рабочих процессов InGit Project это
означает, что даже системы без API могут быть
доступны. Это было за пределами планирования
InGit.

### 2.6. Запланированные Повторяющиеся Задачи

Cowork может запускать запланированные задачи:
еженедельные отчёты, ежедневные проверки
почты, ежемесячные архивы. Определяется
однажды, выполняется автоматически.

Это адресует то, на что дорожная карта InGit
ссылалась как «авто-генерируемые отчёты».

### 2.7. Кросс-Платформенная Доступность

Cowork работает на macOS и Windows (Linux,
вероятно, в конечном итоге). Мобильное
приложение для назначения задач через Dispatch.

План InGit построить Desktop GUI для кросс-
платформенной поддержки становится ненужным,
если Cowork является основным интерфейсом.

### 2.8. Что это удаляет из дорожной карты InGit

Если Cowork обеспечивает вышеуказанное, дорожная
карта InGit может быть существенно сокращена:

**Удалить или отложить**:
- Desktop GUI (Electron + React)
- Web UI для управления проектом
- Мобильное приложение
- Слой постоянной памяти
- Инфраструктура выполнения агента
- Рамка коннекторов
- Система авто-генерации отчётов

**Сохранить и сосредоточиться на**:
- Конвенции файловой структуры
- YAML метаданные как код
- Git-нативное версионирование
- Хуки валидации
- Шифрование для чувствительных данных
- Инструменты миграции (из других систем)
- MCP сервер для операций InGit

Это сокращает InGit от полного приложения до
**специализированной подложки**. Подложка —
гораздо более достижимый объём для
индивидуального разработчика или малой
команды.
