> Источник: MHTML‑снимок `Вакансии в Anthropic по кластерам - Claude` (корень репозитория). Раздел диалога — 21 учитель индийского йога — спектр между Professional Colleague и Representative Agent.

Это не Тип 1 — потому что профиль не общий для всей профессии. И это не Тип 4 — потому что под-агенты не уникальные для принципала, они общие.

Это новый, шестой тип в типологии. Промежуточный.

Технически — что это такое

В технических терминах: вы предлагаете архитектуру mixture of experts, но не на уровне нейронной сети, а на уровне профессиональной идентичности.

Принципал учитель музыки. Его композитный агент имеет доступ к:

Под-агенту по классической композиции

Под-агенту по джазовой импровизации

Под-агенту по электронной музыке

Под-агенту по педагогике музыки

Под-агенту по теории контрапункта

Под-агенту по русской народной музыке

Под-агенту по техникам аранжировки

Каждый под-агент — это, по сути, Профессиональный Коллега-Агент для своей узкой подспециальности. Композитный агент — это оркестратор этого набора, конфигурированного именно для этого учителя.

Другой учитель музыки имел бы другую конфигурацию — может быть, без джаза, но с этномузыкологией. Третий — с акцентом на детскую музыкальную педагогику.

Сочетание делает каждого учителя уникальным. Под-агенты общие. Композитный агент настраивается на индивидуальный набор.

Связь с метафорой йога

Возвращаясь к вашему примеру: двадцать один учитель йоги.

Каждый из двадцати обучающих учителей — это аналог узкоспециализированного Профессионального Коллеги-Агента. Они общедоступны. Любой ученик мог бы прийти к ним.

Двадцать первый учитель выбирает свой набор из этих двадцати. Этот выбор — его уникальность.

Композитный Агент — это AI-аналог именно этого выбора. AI-инфраструктура, которая позволяет принципалу сконфигурировать свой уникальный набор узкоспециализированных AI-помощников.

Спектр — теперь полный

С учётом этого вашего наблюдения, типология агентов на стороне принципала должна быть пересмотрена. Вместо пяти типов — шесть, и они образуют спектр от общего к уникальному:

Тип 0 — Персональный AI-помощник. Универсальный. Без специализации. Один продукт для всех.

Тип 1 — Профессиональный Коллега-Агент. Одна профессия, один шаблон. «Обучай» для всех учителей.

Тип 1.5 — Подспециализированный Коллега-Агент. Узкая специализация в профессии. Для всех адвокатов по социальному праву, не для всех юристов вообще.

Тип 2 — Композитный Агент Навыков (новый). Принципал имеет уникальный набор специализаций. Композитный агент координирует множество узкоспециализированных под-агентов под этот конкретный набор. Каждый под-агент общий, но сочетание уникально.

Тип 3 — Институциональный Агент. Встроен в институциональные процессы.

Тип 4 — Агент-Сотрудник. Делегированная зона ответственности.

Тип 5 — Представительский Агент. Полностью уникальный для одного индивида. Знает его жизненную траекторию, представляет во внешнем мире.

Композитный Агент — это именно та промежуточная категория, которая объединяет общие узкие компетенции в уникальную для принципала конфигурацию.

Почему это важно архитектурно

Композитный Агент решает критическую проблему, которую ни Тип 1, ни Тип 5 решить не могут.

Тип 1 (Профессиональный Коллега) работает только если профессия стандартизована достаточно, чтобы один общий шаблон был полезен. Учителя школы — да. Юристы — да. Но специалист по математической биологии с интересом к теории графов и опытом в эпидемиологии — нет. Никакой шаблон не покрывает эту комбинацию.

Тип 5 (Представительский Агент) требует уникальной разработки для каждого индивида. Дорого, медленно, не масштабируется.

Композитный Агент позволяет масштабироваться: вы инвестируете один раз в построение узкоспециализированных под-агентов (теория графов, эпидемиология, математическая биология), и потом любой принципал может сконфигурировать свою уникальную комбинацию из существующих под-агентов.

Это mixture-of-experts паттерн, перенесённый из архитектуры нейронных сетей в архитектуру профессиональной поддержки.

Экономика тиражирования меняется

В шестой статье я писал, что Тип 1 имеет красивую экономику: одна инвестиция в профессиональный контекст, тысячи пользователей. Это всё ещё верно для Типа 1.

Композитный Агент имеет более интересную экономику: комбинаторное умножение ценности.

Допустим, есть 100 узкоспециализированных под-агентов в области X. Каждый — отдельная инвестиция, но скромная (узкая область, фокусированная база знаний).

Каждый принципал выбирает 10-20 из этих 100. Количество возможных комбинаций — астрономическое: C(100, 15) ≈ 250 миллиардов.

Это значит: каждая комбинация уникальна, но компоненты тиражируются. Принципал получает уникальную поддержку, но при этом платформа окупается за счёт совместного использования базовых компонентов.

Конкретные примеры в разных профессиях

Чтобы это было не абстракцией, разверну на нескольких конкретных областях.

Учитель музыки. Базовая профессиональная подготовка плюс уникальные специализации. Композитный агент собирает под-агентов: по теории музыки, по конкретному инструменту, по детской педагогике, по конкретному жанру (джаз, классика, фолк), по технологиям записи. Каждый учитель имеет свой набор. Никто не имеет всех ста.

Юрист по социальному праву. Композитный агент: SGB IX, SGB XII, конкретные подспециализации (инвалидность, психическое здоровье, дети, пожилые), процедурное право, конкретные земли (Саксония, Бавария), интерфейс с медицинскими экспертизами. Юрист, специализирующийся на правах детей-инвалидов в Саксонии, имеет конкретную конфигурацию из набора.

Врач общей практики. Композитный агент: каждая система органов как под-агент, плюс под-агенты по типичным сопутствующим заболеваниям, по гериатрии, по педиатрии, по конкретным редким состояниям, по интерпретации лабораторных тестов. Конкретный врач, работающий в основном с пожилыми пациентами в сельской местности, имеет свою конфигурацию.

Программный архитектор. Композитный агент: по конкретным технологиям (Python, Go, Rust), по архитектурным паттернам (microservices, event-driven), по доменам (финтех, медтех, e-commerce), по требованиям (compliance, performance, accessibility). Архитектор, специализирующийся на разработке медицинских систем с фокусом на accessibility, имеет свой набор.

Социальный работник. Композитный агент: по типу клиентов (пожилые, дети, бездомные, мигранты), по типу проблем (зависимости, психическое здоровье, домашнее насилие), по системам поддержки (государственные, НКО, церковные), по правовым вопросам. Каждый социальный работник имеет свой набор по контексту своей работы.

В каждой из этих профессий Тип 1 (общий для профессии) недостаточен — слишком много специализаций. Но Тип 5 (уникальный для индивида) избыточен — потому что узкие специализации общие, не уникальные. Композитный Агент — правильное решение.

Технические компоненты

Архитектурно Композитный Агент состоит из:

Реестр Узкоспециализированных Под-Агентов. Каталог всех существующих узкоспециализированных AI-помощников в данной экосистеме. Каждый под-агент имеет описание: какая специализация, какая глубина, какие границы.

Процесс Конфигурирования Принципала. Помогает принципалу выбрать релевантные под-агенты из реестра. Может быть автоматическим (анализ профиля принципала, рекомендации), полуавтоматическим (диалог с принципалом), или ручным (принципал выбирает сам).
