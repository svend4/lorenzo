## Практические рекомендации для текущего проекта

Для конкретного семидокументного проекта
Nautilus / OKWF, практические следующие шаги:

**Шаг 1 — Закоммитить документы в репозиторий.**
Все семь документов должны быть сохранены в
`nautilus/docs/` как файлы Markdown. Это
создаёт стабильные референсные артефакты.

**Шаг 2 — Создать README, который их связывает.**
README верхнего уровня объясняет отношения
между документами, в каком порядке их читать,
что каждый адресует. Это служит индексом
перекрёстных ссылок бедняка.

**Шаг 3 — Открыть GitHub Discussions.**
Включить Discussions на репозитории Nautilus.
Создать начальные категории: Архитектура,
Реализация, Пилоты, Теория, Сообщество. Это
обеспечивает инфраструктуру тредированных
обсуждений.

**Шаг 4 — Создать начальные треды обсуждений.**
Для каждого документа создать один тред
Обсуждения, приглашающий комментарии. Это
делает документы обсуждаемыми.

**Шаг 5 — Использовать Anthropic Project для
текущей работы.** Создать Project Nautilus в
Anthropic. Загрузить документы как Project
Knowledge. Использовать этот Project для
AI-вспомогательной разработки новых аспектов
без потери контекста.

**Шаг 6 — Перекрёстные ссылки неформально.**
При цитировании одного документа из другого
использовать ясные ссылки markdown с путями к
файлам. Принять, что они могут ломаться при
переструктурировании; потеря приемлема.

**Шаг 7 — Отслеживать эволюцию в CHANGELOG.**
Каждый документ должен иметь changelog,
отмечающий крупные ревизии. Это сохраняет
историю без требования сложного
версионирования.

**Шаг 8 — Периодический снимок для потомков.**
Когда достигнуты естественные вехи, тегировать
состояние репозитория. Это создаёт
восстанавливаемые точки.

Это не Слой B. Это **наилучшее доступное
приближение, использующее текущие инструменты**.
Это будет работать. У этого будет трение.
Трение будет ниже, чем без инфраструктуры
вообще.
