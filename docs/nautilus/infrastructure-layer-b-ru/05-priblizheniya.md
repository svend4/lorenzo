# Существующие приближения

> Источник: MHTML‑снимок `Вакансии в Anthropic по кластерам - Claude` (корень репозитория). Раздел диалога — paper «Infrastructure for AI-Collaborative Intellectual Work (RU)».

## Существующие приближения

Несколько существующих инструментов приближают
части того, чем должен быть Слой B. Никакой не
покрывает полную потребность.

### Anthropic Projects

Обеспечивает постоянный контекст через
множественные чаты, сгруппированные вокруг
темы. Наиболее близко к Слою B со стороны чата.
Ограничения:
- Каждый чат внутри проекта по-прежнему
линейный
- Нет тредов или ветвления из конкретных
сообщений
- Нет аннотаций на документах
- Документы хранятся как файлы в Project
Knowledge, но не версионируются способом,
полезным для интеллектуальной работы
- Перекрёстные ссылки между элементами
содержания не первоклассны

Projects **необходимы, но недостаточны**. Они
решают часть проблемы (постоянный контекст), но
не потребности тредов и структуры.

### GitHub Discussions

Обеспечивает тредированный форум внутри
репозиториев GitHub. Тред работает хорошо.
Ограничения:
- Культурно кодо-ориентированный;
интеллектуальные обсуждения чувствуют себя
не на своём месте
- Нет AI-сотрудничества как нативной операции
- Обсуждения отделены от документов в
репозитории; ссылки между ними — внешние
ссылки, не первоклассны
- Нет аннотаций на тексте документа

GitHub Discussions **структурно близок**, но
**культурно далёк** для интеллектуальной работы.

### Notion / Obsidian / Roam

Управление личными знаниями со связанными
заметками. Перекрёстные ссылки работают хорошо.
Ограничения:
- Спроектированы для индивидуального
использования; сотрудничество добавлено как
функция, не ядро
- AI-интеграция добавлена как функция, не
ядро
- Нет нативной модели тредов — связывание
двунаправленное, но не тредированное
- Версионирование документов работает, но не
является центральным организующим
принципом

Эти инструменты **хороши для личных знаний**,
но **не спроектированы для устойчивого
AI-сотрудничества над общей работой**.

### Discourse / phpBB-стиль форумы

Зрелые инструменты тредированных обсуждений.
Тред — центральный организующий принцип.
Ограничения:
- Нет AI-сотрудничества
- Не спроектированы для документов как
первоклассных артефактов
- Не спроектированы для проектно-стилевого
отслеживания работы

Эти инструменты **хороши для общественного
обсуждения**, но **не спроектированы для
развития интеллектуального проекта**.

### Комбинированные рабочие процессы

Большинство серьёзных интеллектуальных работников
комбинируют инструменты:
- Anthropic Projects для AI-вспомогательного
письма
- GitHub для хранения и версионирования
- Notion или Obsidian для личных знаний
- Email или мессенджер для реального времени
- Discord или Slack для сообщества

Это работает. Но фрагментация имеет издержки:
- Контекст разделён через системы
- Перекрёстные ссылки в основном ментальные,
не технические
- Каждая система имеет разные элементы
управления доступом
- Миграция между системами требует усилий
- Каждая система имеет свой UI для изучения

Фрагментация — это то, что автор определял как
проблему.
