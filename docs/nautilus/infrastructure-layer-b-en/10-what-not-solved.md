## What This Document Doesn't Solve

This document identifies a problem. It does not propose a 
solution.

It does not specify Layer B in detail.
It does not provide implementation roadmap.
It does not commit to building anything.
It does not predict which existing player will fill the gap.

It documents the problem so it can be referenced. It surveys 
existing approximations so they can be combined intelligently. 
It points at the recursive possibility (Nautilus extending 
itself) without committing to it.

This is intentional. The seven preceding documents were 
architectural specifications proposing concrete things. This 
one is an inquiry document, naming a missing layer without 
specifying what should fill it.

If the author or readers want to develop solutions, this 
document provides framing. It does not provide answers.
