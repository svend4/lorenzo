## Митигации

### 1. Одиночный разработчик — bus factor 1

**Уровень:** 🔴 КРИТИЧЕСКИЙ (score 20)

**Митигация:** Документировать всё; привлечь авторов OSS-компонентов как контрибьюторов

### 2. Авторы компонентов не ответят на запросы

**Уровень:** 🟠 ВЫСОКИЙ (score 15)

**Митигация:** Подготовить альтернативные компоненты; форкнуть при необходимости

### 3. PII-утечки через MCP-инструменты

**Уровень:** 🟠 ВЫСОКИЙ (score 15)

**Митигация:** SENTINEL allowlist обязателен; quarantine для внешних данных

### 4. Лицензия BSL 1.1 (NGT) ограничит коммерческое использование

**Уровень:** 🟠 ВЫСОКИЙ (score 12)

**Митигация:** Использовать только для внутренних нужд или найти MIT-альтернативу

### 5. Высокая стоимость Claude API при масштабировании

**Уровень:** 🟠 ВЫСОКИЙ (score 12)

**Митигация:** Кешировать промпты; использовать Haiku для рутинных задач; батчинг

### 6. Устаревание документации при быстром развитии компонентов

**Уровень:** 🟠 ВЫСОКИЙ (score 12)

**Митигация:** CI-обновление docs; improve_watcher.py; еженедельные дайджесты

### 7. AgentFS не поддерживает concurrent multi-agent доступ

**Уровень:** 🟠 ВЫСОКИЙ (score 12)

**Митигация:** CRDT-синхронизация или lock-файлы на уровне Knowledge OS

### 8. Прототип Knowledge OS займёт дольше запланированного

**Уровень:** 🟠 ВЫСОКИЙ (score 9)

**Митигация:** Начать с минимальным ансамблем: Svyazi + CardIndex + AgentFS

### 9. Зависимость от закрытого API Anthropic

**Уровень:** 🟡 СРЕДНИЙ (score 8)

**Митигация:** Абстрагировать через провайдер-интерфейс; поддержать Ollama/local LLM

### 10. Конкурирующие OSS-проекты могут обогнать

**Уровень:** 🟡 СРЕДНИЙ (score 6)

**Митигация:** Уникальность — ансамблевый подход; интеграция > замена
