## What's Missing — Layer B

Between chat and repository, there should exist Layer B: an 
intellectual workspace for project development.

What would Layer B look like, ideally?

It would maintain the **fluid creativity** of chat while adding 
the **structural stability** of repository. It would allow 
**branching exploration** without losing context. It would 
preserve **specific points** that can be revisited and developed 
further. It would support **AI collaboration** as a first-class 
operation, not as an external tool.

Concretely, Layer B would offer:

**Threading and branching.** From any message in a discussion, 
the user could create a branch — a sub-thread that explores 
an idea further while preserving the parent. Branches could 
be merged back, abandoned, or remain as alternate developments.

**Annotation on documents.** Specific sections of documents 
could be annotated with comments. Comments could be threaded. 
Other AI agents (or human collaborators) could respond to 
comments. Annotations would persist across document versions.

**Cross-references.** Documents and discussion threads could 
link to each other explicitly. A reference from Document 7 to 
Document 6 would be a real link, not just text. Following the 
link would show context — the surrounding section in the target.

**Versioning without code-orientation.** Documents could be 
versioned, with diffs shown, but without the cultural overhead 
of pull requests and code review processes. Edits could be 
collaborative, with AI assistance integrated.

**AI collaboration as primary operation.** Each thread, each 
document, each annotation could be developed in collaboration 
with AI. The AI would have access to the full context of the 
project — not just the current message, but the entire web 
of related materials.

**Promotion workflow.** Content developed in Layer B could be 
promoted to Layer C (committed to repository) when stable 
enough. Content from Layer A (chat) could be promoted to Layer 
B when worth preserving for further development.

**Bidirectional flow.** Promotion is not one-way. Material 
in Layer C could be brought back to Layer B for further 
development, then re-promoted. Material in Layer B could be 
extracted to chat for fresh exploration with reduced context.

This is the infrastructure that doesn't exist as a unified 
product.
