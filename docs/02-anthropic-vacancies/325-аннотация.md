# Аннотация

## Аннотация

Документ 2.3 определил Слой B — отсутствующую
инфраструктуру между чатом и репозиториями кода
— как фундаментальный пробел в AI-совместной
интеллектуальной работе. Документ был
теоретическим, называющим проблему без
спецификации решений. Этот документ, написанный
со знанием существующего проекта InGit автора и
платформы Cowork от Anthropic (запущенной в
январе 2026), предлагает конкретный путь
реализации.

Мы утверждаем, что InGit и Cowork вместе
существенно закрывают пробел Слоя B, ни один не
достаточен в одиночку. InGit обеспечивает
**структурированную подложку рабочего
пространства** — организацию файлов, YAML
метаданные, Git-нативное версионирование,
шифрование, offline-first работу. Cowork
обеспечивает **слой агентского интеллекта** —
постоянную память, многошаговое выполнение
задач, MCP интеграцию, запланированные задачи,
использование компьютера. Их интеграция создаёт
нечто, что ни один не мог бы предоставить
отдельно: структурированное, версионированное,
AI-усиленное интеллектуальное рабочее
пространство.

Мы предлагаем конкретные пути интеграции в
порядке доступности (от немедленных к
долгосрочным), определяем какие функции InGit
наиболее важны при объединении с Cowork,
уточняем дорожную карту InGit на основе этого
позиционирования, и обсуждаем стратегические
импликации для более широкого проекта
Nautilus / OKWF.

Центральное прозрение: InGit не должен
конкурировать с Cowork. InGit должен
позиционироваться как **слой структуры,
необходимый Cowork** — подложка для устойчивой,
организованной, восстанавливаемой, разделяемой
AI-совместной работы.
