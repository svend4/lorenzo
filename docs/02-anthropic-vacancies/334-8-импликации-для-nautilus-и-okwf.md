## 8. Импликации для Nautilus и OKWF

За пределами InGit конкретно, это
позиционирование имеет импликации для более
широкого проекта Nautilus / OKWF.

### 8.1. Nautilus как Подложка Федерированных Знаний

Nautilus был спроектирован для федерированных
знаний через множественные репозитории с
семантической организацией (Q6 гиперкуб,
Pattern Library). С InGit как подложкой Cowork,
роль Nautilus проясняется:

**Nautilus работает на один уровень выше InGit**.
В то время как InGit структурирует
индивидуальные проекты, Nautilus федерирует
через многие проекты. Множественные InGit
проекты могут быть узлами Nautilus; Nautilus
Portal Protocol включает запросы через них.

Пользователи Cowork с InGit Projects могли бы
опционально федерировать через Nautilus, получая
кросс-проектный поиск, разделение паттернов и
пути вклада.

Это согласовано с Документом 1 (Portal
Protocol), но теперь с конкретной подложкой
внизу.

### 8.2. Импликации для OKWF Пилота

Документ 4 (Концепт OKWF) и Документ 6
(Профессиональные Коллеги-Агенты) предложили
пилот SGB Адвоката-Коллеги. С доступной
комбинацией InGit/Cowork:

**Архитектура пилота уточнена**:
- Каждый адвокат имеет InGit Project для
  своей работы по делам
- Cowork работает в этих Projects
- MCP сервер для SGB-специфичных знаний
  (статуты, прецеденты)
- Интеграция Pattern Library через Nautilus
- Кросс-адвокатное разделение через
  федерированный Nautilus

Это драматически более достижимо, чем
строительство всего на заказ. Существенная
инфраструктура (Cowork) используется.
Специализированные слои (SGB MCP сервер,
библиотека паттернов) — фокусированные
дополнения, не вся система.

**Пересмотренный бюджет Год 1**: Оригинальная
оценка Документа 6 была €430К, затем Документ
7 увеличил до €1,5М для композитной
архитектуры. С Cowork как агентским слоем:

- Cowork: включён в подписки пользователей
  (€20/месяц на практикующего)
- Подложка InGit: ~€100К для созревания
- SGB-специфичный MCP сервер: ~€200К для
  построения
- Интеграция библиотеки паттернов: ~€100К
- Операции и поддержка: ~€100К
- **Пересмотренный Год 1: €500К**, плюс
  подписки Cowork практикующих

Ближе к оригинальной оценке, более достижимо,
использует существующую инфраструктуру.

### 8.3. Композитный Агент Навыков в этой Архитектуре

Документ 7 представил Композитный Агент
Навыков — Тип 1.5 между Профессиональным
Коллегой (Тип 1) и Представительским Агентом
(Тип 4). С Cowork:

**Cowork — это координатор композитных
агентов**. Конфигурация sub-агентов
(специализированные MCP серверы для узких
специализаций) — это композитная идентичность
принципала. Cowork оркестрирует их, выводит на
поверхность разногласия, интегрирует выходы.

Sub-агенты становятся специализированными MCP
серверами:
- `mcp-sgb-ix-section-78`
- `mcp-german-procedural-law`
- `mcp-saxony-court-procedures`
- `mcp-ksv-sachsen-patterns`
- `mcp-medical-assessment-interpretation`

Каждый независимо поддерживается. Каждый может
быть добавлен в Cowork Project практикующего
по необходимости. Конфигурация — буквально
набор MCP серверов, активных для этого Project.

Это более конкретно, чем абстрактное описание
Документа 7. Архитектура отображается на
реальные способности Cowork.

### 8.4. Рекурсивное Прозрение Документа 2.3 Реализовано

Документ 2.3 отметил рекурсивную симметрию:
Nautilus, расширяющий себя для обеспечения
инфраструктуры для собственной разработки. С
InGit/Cowork:

**Это теперь имеет конкретную форму**.
Интеллектуальный проект Nautilus / OKWF может:
- Жить в InGit Project (подложка)
- Разрабатываться в Cowork (агентский слой)
- Федерироваться через Nautilus (слой через
  проекты)
- Информировать собственную архитектуру через
  использование

Рекурсивное прозрение больше не аспирационное.
Оно реализуемо начиная со следующей недели.
