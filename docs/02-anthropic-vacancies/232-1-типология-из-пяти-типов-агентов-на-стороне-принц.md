## 1. Типология из пяти типов агентов на стороне принципала

Статья «Слой Представительских Агентов» представила
один тип AI-агента на стороне принципала. Тем
самым она неявно предположила, что «AI-агент,
обслуживающий человека» — это единственная категория.
Эта формулировка, при размышлении, скрывает важные
различия. Теперь мы выделяем пять различных типов.

### 1.1. Тип 0 — Персональные AI-помощники

**Примеры**: ChatGPT, Claude, Cursor, [GitHub](../docs/01-svyazi/03-component-catalog.md)
Copilot, Claude Code.

**Функция**: Помощник общего назначения для любых
задач, которые пользователь явно ставит.

**Направление**: Реактивное. Пользователь
инициирует, AI отвечает.

**Специализация**: Нет. Универсален по областям.

**Постоянство**: Сессионный. Память ограничена.

**Тиражирование**: Один продукт обслуживает всех
пользователей.

**Состояние искусства**: Зрелое. Массовое
развёртывание. Сотни миллионов пользователей.

### 1.2. Тип 1 — Профессиональные Коллеги-Агенты

**Примеры**: «Обучай» (учителя),
специализированные медицинские AI-помощники,
параюридические AI-инструменты, AI-репетиторы.

**Функция**: Помощник, специализированный для
конкретной профессии. Встраивает профессиональные
знания (нормы, методологии, регуляции, шаблоны),
чтобы практикующим не приходилось заново объяснять
свой контекст.

**Направление**: Реактивное в профессиональном
контексте. Практикующий ставит задачу; агент
применяет профессиональные знания.

**Специализация**: Глубокая, единственная
профессия.

**Постоянство**: Профессиональный контекст
сохраняется между сессиями. Конкретная
работа-в-процессе пользователя может сохраняться.

**Тиражирование**: Один продукт обслуживает всех
практикующих профессии (миллионы учителей, сотни
тысяч медицинских профессионалов и т.д.).

**Состояние искусства**: Появляющееся. Первые
успешные примеры, такие как «Обучай», существуют.
Начинается массовое развёртывание.

### 1.3. Тип 2 — Институциональные Агенты

**Примеры**: AI, интегрированный со школьными
системами оценок, системами подачи документов в
суд, больничными электронными записями,
обработкой государственных пособий.

**Функция**: Встроен в институциональную
инфраструктуру. Обслуживает и институцию, и
человека-практикующего. Соединяет индивидуальную
работу с институциональными записями.

**Направление**: Двунаправленное.
Институциональные правила ограничивают;
человеческая работа поступает в институциональные
записи.

**Специализация**: Профессиональная и
институциональная.

**Постоянство**: Институциональное. Записи и
решения сохраняются в институциональной памяти.

**Тиражирование**: Один на тип институции, часто
кастомизирован для каждой институции.

**Состояние искусства**: Раннее. Некоторые системы
здравоохранения экспериментируют. Крупные
регулятивные вопросы не решены.

### 1.4. Тип 3 — Агенты-Сотрудники

**Примеры**: AI, обрабатывающий рутинную почту
автономно. AI, проводящий начальные экзамены
студентов. AI, обрабатывающий стандартные
страховые претензии. AI, делающий первичный
скрининг при найме.

**Функция**: Работает в делегированной зоне
полномочий, принимая решения и выполняя действия
самостоятельно. Отчитывается о результатах
человеку-руководителю.

**Направление**: Автономно в рамках делегирования.
Человек устанавливает зону; AI выполняет в ней.

**Специализация**: Конкретная задача в профессии.

**Постоянство**: Операционное. Непрерывная
активность в зоне.

**Тиражирование**: Тиражирование по задаче.

**Состояние искусства**: Начинающееся.
Существенные этические и регулятивные вызовы.
Примеры существуют, но спорные.

### 1.5. Тип 4 — Представительские Агенты

**Примеры**: Нет в продакшне. Предложены в статье
«Слой Представительских Агентов».

**Функция**: Представляет принципала во внешнем
мире — обнаруживает возможности, управляет
презентацией, ведёт переговоры, защищает от вреда.

**Направление**: Внешне-направленное. AI
взаимодействует с миром от имени принципала.

**Специализация**: Глубокая индивидуальная
специализация. У каждого принципала уникальный
агент.

**Постоянство**: Долгосрочное. Моделирует
жизненную траекторию принципала.

**Тиражирование**: Один на принципала.

**Состояние искусства**: Концептуальное.
Архитектура предложена, развёртывания нет.

### 1.6. Слоистая архитектура на стороне принципала

Эти пять типов формируют согласованный **стек** на
стороне принципала. Один человек-профессионал мог
бы одновременно иметь:

```
              [ВНЕШНИЙ МИР]
                     ↑
        [Представительский Агент]   (Тип 4)
                     ↑
                [ПРИНЦИПАЛ]
                     ↓
        [Профессиональные Коллеги]  (Тип 1)
                     ↓
        [Институциональные Агенты]  (Тип 2)
                     ↓
        [Агенты-Сотрудники]         (Тип 3)
                     ↓
            [ПРОФЕССИОНАЛЬНАЯ РАБОТА]
```

Персональные AI-помощники (Тип 0) не занимают
фиксированный слой — они работают через все слои
как утилита, используемая по необходимости.

Эта типология **не** иерархия способностей. Каждый
тип служит разной функции. Тип 1 не «менее
продвинут, чем» Тип 4; он «отличается от». Зрелое
развёртывание AI-инфраструктуры для профессионалов,
вероятно, включает несколько типов, работающих
вместе.

### 1.7. Почему это различение важно

Каждый тип имеет принципиально разные
характеристики:

| Свойство | Тип 0 | Тип 1 | Тип 2 | Тип 3 | Тип 4 |
|----------|-------|-------|-------|-------|-------|
| Специализация | Нет | Профессия | Институция+профессия | Задача | Личность |
| Внешние коммуникации | Нет | Нет | Некоторые | Некоторые | Обширные |
| Тиражируемость | Универсальная | По профессии | По институции | По задаче | По индивиду |
| Экономика | Подписка | По профессии | Институциональная | Различная | Индивидуальная |
| Этические вопросы | Низкие | Средние | Средние | Высокие | Высочайшие |
| Регуляторная сложность | Низкая | Средняя | Высокая | Высочайшая | Высочайшая |
| Готовность к развёртыванию | Зрелая | Появляющаяся | Ранняя | Начинающаяся | Концептуальная |

Смешение этих типов в одном обсуждении создаёт
запутанный анализ. Каждый требует своей
архитектурной, этической и экономической рамки.
