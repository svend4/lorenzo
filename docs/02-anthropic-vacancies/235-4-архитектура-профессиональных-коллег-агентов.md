## 4. Архитектура Профессиональных Коллег-Агентов

### 4.1. Трёхслойная внутренняя архитектура

Профессиональный Коллега-Агент имеет три
различных внутренних слоя:

**Слой A — База Профессиональных Знаний.**
Закодированное знание профессии: стандарты,
регуляции, методологии, шаблоны, словари, общие
паттерны. Курируется и поддерживается
профессиональными экспертами (с возможной
помощью AI для обновлений).

Это самый дорогой слой для построения и наиболее
ценный конкурентный ров. Высококачественная база
профессиональных знаний для немецкого социального
права, например, представляет собой тысячи часов
экспертной кураторской работы.

**Слой B — Движок Генерации и Рассуждений.**
Активный компонент: LLM (или ансамбль LLM),
сконфигурированный для работы в Слое A.
Принимает ввод практикующего, применяет
профессиональные знания, генерирует
соответствующий результат.

Этот слой в значительной степени взаимозаменяем.
Разные провайдеры LLM могут питать один и тот же
Слой A.

**Слой C — Интерфейс Практикующего и
Персонализация.** Видимое приложение:
веб/мобильный/десктопный интерфейс,
аутентификация, личные предпочтения, хранение
работы-в-процессе, интеграция с другими
инструментами практикующего.

Персонализация здесь не доходит до Слоя A —
профессиональный контекст общий для всех
пользователей. Личные предпочтения (UI, язык,
настройки уведомлений, текущие проекты)
наслаиваются сверху.

### 4.2. Стратегии курирования знаний

Слой A — база профессиональных знаний — это
сердце Профессионального Коллеги-Агента.
Несколько стратегий для его построения и
поддержки:

**Стратегия 1 — Один эксперт-куратор.** Старший
профессионал или небольшая команда курирует базу
знаний. Высочайшее качество, ограниченный
масштаб. Хороша для ранних стадий.

**Стратегия 2 — Федерация практикующих.** Многие
практикующие вносят фрагменты знаний через
механизм библиотеки паттернов. Качество
варьируется, масштаб большой. Требует
кураторского надзора. Это связывает напрямую с
архитектурой Библиотеки Паттернов в Nautilus.

**Стратегия 3 — Интеграция с авторитетными
источниками.** Прямая интеграция с
авторитетными источниками (юридические базы
данных, медицинские рекомендации, образовательные
стандарты). Автообновление при обновлении
источников. Наиболее строго, требует
правового/институционального доступа.

**Стратегия 4 — Гибридная.** Комбинация
авторитетных источников (для жёстких правил) и
паттернов от практикующих (для практических
применений). Наиболее практична для сложных
профессий.

Для большинства областей Стратегия 4 реалистична.
Сервис «Обучай», по-видимому, использует её —
официальный ФГОС для жёстких правил, уточнения от
практикующих для практического применения.

### 4.3. Обновление и версионирование

Профессиональные знания меняются. Налоговые
кодексы обновляются. Медицинские рекомендации
пересматриваются. Образовательные стандарты
модифицируются. Юридические прецеденты
сдвигаются.

Профессиональные Коллеги-Агенты должны
обрабатывать это, не ломая рабочие процессы
практикующих. Два принципа:

**Принцип Стабильности Снапшота.**
Работа-в-процессе практикующего привязана к
конкретной версии базы знаний. Обновления базы
знаний в середине проекта не изменяют ретроспективно
черновики практикующего. Это предотвращает
дезориентацию.

**Принцип Осведомлённости об Обновлениях.**
Существенные обновления базы знаний выводятся на
поверхность практикующему. «Соответствующий
статут был изменён на прошлой неделе. Ваше
текущее обращение может потребовать пересмотра.»
Практикующий решает, обновлять ли свою работу.

Эти принципы параллельны практикам разработки
программного обеспечения (неизменяемые релизы,
уведомления об устаревании). Профессиональные
Коллеги-Агенты наследуют эти паттерны.

### 4.4. Архитектура Обеспечения Качества

Профессиональные результаты имеют последствия.
Дефектный юридический документ, медицинская
запись или образовательная оценка производит
реальный вред. Обеспечение качества должно быть
встроено в архитектуру.

**Механизм QA 1 — Индикаторы уверенности.** Агент
сообщает о своей уверенности в каждом элементе
результата. Высокая уверенность: «Стандартный
формат применён согласно текущей регуляции.»
Низкая уверенность: «Подход неопределён;
консультируйтесь с руководством специалиста.»

**Механизм QA 2 — Цитированные источники.**
Каждое существенное утверждение, которое агент
делает, прослеживается до источника в базе
знаний. Практикующий может проверить, следуя
цитатам.

**Механизм QA 3 — Требуется проверка
практикующим.** Результаты — это черновики,
требующие проверки и принятия практикующим, не
окончательные продукты. Интерфейс агента должен
делать проверку естественной, а не
обременительной.

**Механизм QA 4 — Сообщение об ошибках.**
Практикующие, столкнувшиеся с ошибками, имеют
способ сообщить о них с низким трением.
Агрегированные отчёты информируют улучшения Слоя A.

Эти механизмы вместе производят эквивалент
внимательного коллеги: присутствующего, полезного,
прослеживаемого, погрешимого, но явного об этом.

### 4.5. Стандарты интеграции

Профессиональные Коллеги-Агенты должны
интегрироваться с:

**Стандартными форматами данных** для профессии.
Юридические документы должны выводиться в
стандартных форматах (обычно структурированный
XML для некоторых юрисдикций, форматированный
PDF/DOCX в других). Медицинские записи должны
выводиться в клинических стандартах (HL7, FHIR).
Образовательные материалы в стандартных форматах
обмена.

**Рабочими инструментами**, которые использует
профессия. Юристы используют конкретные системы
управления делами. Учителя используют конкретные
системы электронных журналов. Врачи используют
конкретные системы EHR. Интеграция здесь убирает
трение.

**Системами авторитета**, к которым привязана
профессия. Юридические подачи в судебные
системы. Налоговые подачи в правительство.
Образовательные сертификации. Агент должен
способствовать взаимодействию с авторитетами, а
не создавать новые параллельные.

Эта работа по интеграции значительна. Это также
то, где многие проекты «AI для X профессии»
проваливаются — построив прекрасного агента,
которого ни один практикующий не может вписать в
свой существующий рабочий процесс.
