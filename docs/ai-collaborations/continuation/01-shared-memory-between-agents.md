# Общая память между агентами (CoAlly + ансамбль F)

> Источник: MHTML‑снимок `Поиск коллабораций AI проектов` (корень репозитория).

1. Новый важный слой: общая память между агентами, а не только память одного агента

В предыдущей карте у нас уже были Yodoca, NGT Memory, AgentFS, mclaude и knowledge-space. Но теперь в общую схему стоит добавить ещё один близкий проект — CoAlly из статьи “Контекстная амнезия: три агента, три IDE, ноль общей памяти”. Его ключевая идея: разные агенты и IDE не должны заново учиться одному и тому же контексту; система сохраняет архитектурные решения, баги, root cause analysis, commit hash, полезные знания и делает их доступными через общий MCP‑слой. Автор прямо противопоставляет это статичным CLAUDE.md, .cursorrules и ручному Memory Bank; CoAlly использует семантический поиск, локальные multilingual embeddings, асинхронную векторизацию, on‑premise режим, изоляцию организаций, envelope encryption и поддержку Claude Code, Cursor, Windsurf, Continue, Gemini CLI и других MCP‑совместимых клиентов. Habr

Для Svyazi‑2.0 это важнее, чем кажется. Svyazi в исходном виде умеет строить карточки людей и находить связи между участниками; CoAlly‑подобный слой переносит эту идею на агентов и команды агентов: кто что уже выяснил, какие гипотезы проверены, какие решения приняты, какие ошибки повторяются. Вместе это даёт новый тип сущности: не только person_card, project_card, episode_card, но и agent_experience_card.

Ансамбль F — Cross‑Agent Memory Bus

Родители: Svyazi + CoAlly + AgentFS + Yodoca/NGT Memory + mclaude.

Что рождается: единая шина памяти, где люди, проекты, агенты и решения живут в одном пространстве карточек.

Mermaid

Новое свойство здесь — память становится не персональной, а коллективной и межагентной. Один агент нашёл, что кандидат A хорошо подходит к проекту B; другой агент проверил evidence; третий агент нашёл риск; четвёртый агент не начинает с нуля, а видит уже накопленную историю. mclaude дополняет это locks/handoffs/mailbox‑механикой: несколько Claude‑сессий могут писать handoffs, брать locks, обмениваться сообщениями и работать над общим состоянием без постоянного участия человека. В описанном кейсе mclaude использовался 3 людьми и 6 параллельными сессиями Claude Code, с 27 handoffs за 4 дня и без повторённых тупиков. Habr

Для Svyazi‑2.0 это означает: карточка коллаборации должна учитывать не только людей, но и историю агентных проверок. Например: “эти двое потенциально подходят друг другу” — это не финальный вывод, а объект с историей: кто предложил, какой evidence, кто проверил, какие сомнения, что устарело, что подтверждено.
