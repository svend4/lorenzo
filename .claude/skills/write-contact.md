# write-contact

Помогает написать первое сообщение автору проекта.

## Когда использовать
Когда пользователь хочет написать автору OSS-проекта:
- "Напиши письмо автору AgentFS"
- "Составь запрос на коллаборацию для kksudo"
- "Мне нужно написать spbmolot"
- "Подготовь сообщение для автора Yodoca"

## Шаги

1. **Определить проект и автора**
   - Если не указан → предложить список из `docs/CONTACTS.md`
   - Если указан → найти в `docs/contacts/<slug>.md`

2. **Прочитать контактный файл**
   ```
   Read: docs/contacts/<slug>.md
   ```
   Взять: профиль, слой, открытые вопросы, черновик первого сообщения.

3. **Прочитать исходный файл проекта**
   ```bash
   find docs/ -iname "*{project}*" | head -3
   ```
   Взять: конкретные технические детали для персонализации.

4. **Уточнить у пользователя** (если не очевидно):
   - Какова цель контакта: коллаборация / вопрос / знакомство?
   - Есть ли конкретный вопрос?
   - Платформа: Habr / GitHub / Telegram?

5. **Сгенерировать персонализированное сообщение**

   Требования к сообщению:
   - Конкретная ссылка на проект (название, URL из файла)
   - Один конкретный технический вопрос (из поля "Первый вопрос" или CONTACTS.md)
   - Краткое объяснение Svyazi в 1 предложении
   - Указание конкретной синергии (не абстрактно)
   - Длина: 5-7 предложений максимум

6. **Обновить статус в контактном файле**
   После одобрения пользователем — обновить чеклист в `docs/contacts/<slug>.md`:
   ```
   - [x] Изучили профиль
   ```

## Шаблон сообщения

```
Здравствуйте, {Имя}!

Изучил ваш проект {Проект} на Хабре — особенно заинтересовал {конкретная деталь}.

Я собираю {Svyazi 2.0} — локальную community intelligence platform,
где {Проект} закрыл бы слой {слой}.

Вопрос: {конкретный технический вопрос из CONTACTS.md}.

Было бы интересно пообщаться — как лучше связаться?
```

## Важно

- Не использовать общие фразы ("интересный проект", "хотел бы сотрудничать")
- Конкретный вопрос обязателен — без него не отправлять
- Сообщение на русском (авторы с Habr)
- Максимум один вопрос в первом сообщении
