# synthesize

Синтез единой картины по теме из многих документов.

## Когда использовать

- "сделай синтез по теме X"
- "собери всё про Y в одну картину"
- "что мы знаем о Z в целом"
- "что говорят все источники про W"

## Шаги

1. **Собрать релевантные документы**
   ```bash
   python scripts/improve_passage_retrieval.py --query "тема" --top 20
   python scripts/improve_reading_list.py --query "тема" --top 15
   ```

2. **Прочитать все (или хотя бы топ-10)**
   - Использовать существующие резюме из `docs/SUMMARIES.md`
   - Для каждого взять `<!-- summary -->` + первые 30 строк

3. **Декомпозировать тему на подтемы**

   Стандартные оси разреза:
   - **Уровень:** концептуальный / архитектурный / реализационный
   - **Слой:** memory / knowledge / orchestration / ingestion
   - **Зрелость:** идея / спецификация / прототип / production
   - **Источник:** ваш / Хабр / Anthropic / ChatGPT / academic

4. **Составить матрицу**

   | Подтема | Что сказано | Кем | Где |
   |---------|-------------|-----|-----|
   | [подтема A] | [тезис 1] | [автор] | [`docs/...md`] |

5. **Найти**
   - **Соглашения:** на чём все сходятся
   - **Различия:** в чём противоречия
   - **Пробелы:** о чём не написано
   - **Эволюцию:** как менялись подходы (по датам)

6. **Структура итогового синтеза:**

   ```markdown
   # Синтез: [Тема]

   ## TL;DR
   [3-5 строк главного]

   ## Что мы знаем (consensus)
   - [тезис 1] — поддерживают: [файлы]
   - [тезис 2]

   ## Где разногласия
   - [вопрос 1]:
     - Позиция A: [...]
     - Позиция B: [...]

   ## Чего не хватает
   - [пробел 1]
   - [пробел 2]

   ## Эволюция понимания (хронология)
   - 2024-XX: [понимание 1]
   - 2025-XX: [понимание 2]
   - 2026-04: [текущее]

   ## Открытые вопросы
   1. [вопрос]
   2. [вопрос]

   ## Источники
   - [`docs/path/file1.md`](...)
   - ...
   ```

7. **Опционально: создать постоянный артефакт**
   ```bash
   python scripts/improve_template_init.py --type research-note \
     --slug docs/syntheses/synth-NNN.md \
     --vars title="Синтез: ..."
   ```

## Антипаттерны
- ❌ Перечисление без структуры — синтез это структура
- ❌ Без атрибуции — каждый тезис должен иметь источник
- ❌ Игнорирование дат — игнорирует эволюцию подхода
- ❌ Без открытых вопросов — значит не подумали
