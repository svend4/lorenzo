# audit-corpus

Сводный аудит состояния всего монорепо: метрики, противоречия, пробелы.

## Когда использовать

- "оцени состояние репо"
- "что сейчас с базой знаний"
- "покажи health check"
- "где проблемы в документах"
- "регулярный аудит"

## Шаги

1. **Прочитать сводку**
   ```bash
   Read: docs/HEALTH.md
   Read: docs/SCORING.md
   Read: docs/METRICS.md
   ```

2. **Извлечь ключевые показатели**
   - Общий балл (`/100`)
   - Тренд (по сравнению с предыдущим снапшотом)
   - Зоны 🔴 (требуют действий)

3. **Параллельно проверить:**
   - Битые ссылки → `docs/BROKEN_LINKS.md`
   - Сиротские файлы → `docs/ORPHANS.md`
   - Противоречия → `docs/CONTRADICTIONS.md`
   - Пробелы → `docs/CONTENT_GAPS.md`
   - Свежесть → `docs/STALENESS.md`
   - Дубли → `docs/DUPLICATES.md`

4. **Сформировать отчёт**

   ```markdown
   # Аудит корпуса (дата)

   **Общий балл:** XX/100 ([↑ +N | ↓ -N])

   ## 🔴 Критические проблемы
   - [Проблема 1]: [значение]/[порог]
   - [Проблема 2]: ...

   ## 🟡 Требуют внимания
   - [Метрика 1]: ...

   ## 🟢 В норме
   - [Метрика 1]
   - [Метрика 2]

   ## Топ-5 действий
   1. [действие] — затронет [метрику X]
   2. ...

   ## Команды для исправления
   - `python scripts/improve_X.py` — закроет [проблему]
   ```

5. **Если запрашивают сравнение во времени:**
   ```bash
   Read: docs/KPI_HISTORY.md
   ```

6. **Опционально: создать снапшот**
   ```bash
   python scripts/improve_kpi_snapshot.py
   python scripts/improve_template_init.py --type kpi-snapshot \
     --slug docs/kpi-snapshots/$(date +%Y-%m-%d).md
   ```

## Чеклист аудита

- [ ] Балл HEALTH ≥ 80 ?
- [ ] Битых ссылок 0 ?
- [ ] Противоречий со статусом `open` < 5 ?
- [ ] Пробелов content gaps < 10 ?
- [ ] Свежесть: 90% файлов обновлены за последние 30 дней ?
- [ ] Покрытие тегами ≥ 90% ?
- [ ] Дубли = 0 ?
