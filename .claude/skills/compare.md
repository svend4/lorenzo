# compare

Сравнение двух или нескольких документов / разделов / подходов.

## Когда использовать

- "сравни X и Y"
- "в чём разница между A и B"
- "что общего у X и Y"
- "какой подход лучше: A или B"
- "сопоставь две версии документа"

## Шаги

1. **Определить объекты сравнения**
   - Два файла: пути
   - Два раздела: пути к папкам
   - Два подхода: найти представляющие документы через `search`-скилл
   - Две версии: использовать `git log` для исторических

2. **Выбрать инструмент**

   | Случай | Команда |
   |---|---|
   | Два конкретных файла | `python scripts/improve_compare_docs.py --a docs/X.md --b docs/Y.md` |
   | Версии файла | `git diff HEAD~5 -- docs/X.md` |
   | Семантический diff проекта | `python scripts/improve_version_diff.py --last 10` |
   | Дубли между папками | `python scripts/improve_duplicate_across.py --internal` |
   | Похожие абзацы | `cat docs/SIMILAR_PASSAGES.md` |

3. **Структура сравнения**

   Делать матрицей, не текстом:

   | Параметр | X | Y |
   |---|---|---|
   | Цель | … | … |
   | Подход | … | … |
   | Сильные стороны | … | … |
   | Слабые стороны | … | … |
   | Зрелость | … | … |
   | Лицензия | … | … |
   | Применимость к Svyazi | … | … |

4. **Дать рекомендацию**
   - **Когда X лучше:** [условия]
   - **Когда Y лучше:** [условия]
   - **Гибрид возможен?** (если да — как)

5. **Указать пробелы**
   - Чего нет в обоих
   - Что есть в одном, но отсутствует в другом

## Пример

Пользователь: «сравни Yodoca и NGT Memory»

Действия:
1. Найти файлы:
   ```bash
   find docs -iname "*yodoca*"
   find docs -iname "*ngt-memory*"
   ```
2. Прочитать оба
3. Использовать инструмент:
   ```bash
   python scripts/improve_compare_docs.py \
     --a docs/05-habr-projects/memory/yodoca.md \
     --b docs/05-habr-projects/memory/ngt-memory.md
   ```

Ответ:

> **Сравнение: Yodoca vs NGT Memory**
>
> | Параметр | Yodoca | NGT Memory |
> |---|---|---|
> | Слой | memory | memory |
> | Подход | Hot/Slow path, два контура | Гиппокампальная модель, Hebbian/STDP |
> | Лицензия | … | … |
> | Зрелость | … | … |
>
> **Когда лучше Yodoca:** … (низкоуровневые задачи памяти)
> **Когда лучше NGT:** … (необходима имитация биологической пластичности)
>
> **Возможен ли гибрид:** да — Yodoca как hot path, NGT как slow path
>
> **Пробелы в обоих:** ни один не покрывает governance уровня
