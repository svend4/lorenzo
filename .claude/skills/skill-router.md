# skill-router

Мета-скилл: выбирает подходящий специализированный скилл по запросу пользователя.

## Когда использовать

Используется автоматически при любом запросе. Это **первый скилл, который нужно
рассмотреть**: он анализирует запрос и направляет к специализированному.

Триггеры — любой запрос пользователя, для которого не очевиден один скилл.

## Шаги

1. **Прочитать каталог манифестов**
   ```bash
   Read: docs/TASKS_INDEX.md
   ```

2. **Прочитать каталог скилов** (всех файлов в `.claude/skills/`)
   ```bash
   ls .claude/skills/
   ```
   Прочитать первые 30 строк каждого для понимания триггеров.

3. **Сопоставить запрос с триггерами**

   Для каждого скилла проверить trigger_phrases / секцию «Когда использовать».
   Считать score:
   - +5 за точное совпадение фразы
   - +3 за частичное совпадение (одно слово)
   - +1 за тематическую близость

4. **Если есть составной скилл, который покрывает запрос — приоритет ему**

   Пример: запрос «обход в начале дня» → лучше `daily-routine` чем
   `audit-corpus` + `find-gaps` отдельно.

5. **Если нет однозначного победителя — спросить**

   Пользователь: «нужно разобраться с RAG»
   Ответ:
   > Понимаю несколько разных задач:
   > - **`new-research`** — изучить тему RAG с нуля
   > - **`compare`** — сравнить RAG-подходы между собой
   > - **`generate-rfc`** — формализовать выбор RAG-подхода
   > - **`search`** — найти что уже есть в репо про RAG
   >
   > Что ближе к вашей цели?

6. **Запустить выбранный скилл**

7. **Логировать выбор**
   В конце сессии можно записать в `.claude/skill_metrics.jsonl`:
   ```json
   {"ts": "...", "request": "...", "chosen_skill": "...", "alternatives": [...]}
   ```

## Карта решений (decision tree)

| Запрос содержит | Скилл |
|---|---|
| «найди», «ищу», «где есть» | `search` |
| «кратко», «резюме», «TL;DR» | `summarize` |
| «сравни», «отличия», «vs» | `compare` |
| «улучши», «исправь», «качество» | `improve` |
| «напиши автору», «контакт» | `write-contact` |
| «оцени состояние», «health» | `audit-corpus` |
| «противоречия», «конфликты» | `find-contradictions` |
| «не хватает», «gaps» | `find-gaps` |
| «синтез», «общая картина» | `synthesize` |
| «история решений» | `track-decisions` |
| «MVP», «прототип» | `plan-mvp` |
| «RFC», «спецификация» | `generate-rfc` |
| «ансамбль», «комбинация» | `design-ensemble` |
| «стоит ли брать», «оцени технологию» | `evaluate-tech` |
| «мега-стек», «полный стек» | `propose-mega-stack` |
| «архитектурное ревью» | `review-architecture` |
| «cinderella», «недополучают» | `find-cinderella` |
| «daily», «начало дня» | `daily-routine` |
| «weekly», «неделя» | `weekly-review` |
| «outreach», «день аутрича» | `outreach-day` |
| «исследование», «новая тема» | `new-research` |
