# review-docs

Рецензирует документ или раздел базы знаний Lorenzo.
Когда использовать: пользователь просит проверить, улучшить, оценить
качество конкретного файла или раздела docs/.

## Шаги

1. **Прочитать целевой файл**
   ```
   Read: docs/{path}
   ```

2. **Запустить аналитические скрипты** (только нужные)
   ```bash
   # Метрики качества
   python scripts/improve_metrics.py
   
   # Сложность чтения
   python scripts/improve_complexity.py
   
   # Сломанные ссылки (только если есть ссылки в файле)
   python scripts/improve_broken_links.py
   ```

3. **Прочитать результаты**
   - `docs/METRICS.md` — строка с нужным файлом
   - `docs/COMPLEXITY.md` — строка с нужным файлом
   - `docs/BROKEN_LINKS.md` — строки с нужным файлом

4. **Проверить вручную**
   - Есть ли `<!-- summary: -->` комментарий?
   - Есть ли `<!-- tags: -->` комментарий?
   - Есть ли заголовок `## Статус` (от improve_autofill)?
   - Есть ли конкретные факты или только общие слова?
   - Есть ли ссылки на связанные документы?

5. **Сформулировать рецензию** в формате:

   ```
   ## Рецензия: {filename}

   **Балл качества:** {N}/100

   ### Что хорошо
   - {пункт}

   ### Что улучшить
   - [ ] {конкретное исправление с примером}
   - [ ] {конкретное исправление}

   ### Предлагаемые правки
   [конкретный diff или новый текст для нужных секций]
   ```

6. **Предложить автоматические улучшения**
   - Если нет summary → добавить
   - Если нет тегов → предложить теги
   - Если файл слишком короткий → `improve_llm_enrich.py --file`
   - Если много дублей → `improve_dedup.py`

## Критерии качества (из improve_metrics.py)

| Критерий | Хорошо | Плохо |
|----------|--------|-------|
| Есть summary | ✅ | ❌ добавить |
| Есть теги | ✅ | ❌ добавить |
| Ссылок/1K слов | > 5 | < 2 |
| Слов | 200-5000 | < 50 или > 20000 |
| Заголовков | 3-10 | 0 или > 20 |

## Только для Lorenzo-специфичных файлов

Проверить дополнительно:
- Проект упомянут в `docs/ENTITIES.md`?
- Автор есть в `docs/contacts/`?
- Файл связан с другими через `docs/SIMILAR.md`?
