# summarize

Резюмирование документа, раздела или подборки документов по теме.

## Когда использовать

Пользователь хочет краткий пересказ:
- "кратко расскажи о разделе X"
- "сделай резюме файла Y"
- "что в двух словах про Z"
- "суммаризуй последние изменения"
- "сделай TL;DR по теме"

## Шаги

1. **Определить scope**

   | Что просят | Что читать |
   |---|---|
   | Один файл | `Read docs/path/file.md` |
   | Раздел / папка | `find docs/<section> -name "*.md"` + Read README.md раздела |
   | Тема | сначала `python scripts/improve_reading_list.py --query "X"` |
   | Период (последние N дней) | `docs/DIGEST_AUTO.md` или `docs/DIGEST_WEEKLY.md` |

2. **Использовать существующие резюме**
   - `docs/SUMMARIES.md` — TextRank-резюме каждого файла (~291 файл)
   - `<!-- summary -->` в начале файла — ручное резюме автора
   - `<!-- abstract-auto -->` — автоматический абстракт «проблема/подход/результат»

3. **Уровень детализации**
   - **TL;DR** (1 предложение) — главная мысль
   - **Краткое** (3-5 пунктов) — ключевые тезисы
   - **Подробное** (10-15 предложений) — структурированный пересказ с разделами

4. **Структура подробного резюме**
   - **Что это** (1-2 предложения)
   - **Главные идеи** (3-5 пунктов)
   - **Ключевые сущности** (проекты, термины, авторы)
   - **Открытые вопросы** (если есть)
   - **Связанные документы** (1-3 ссылки)

5. **Формат ответа в зависимости от запроса**

   ```markdown
   **TL;DR:** [одно предложение]

   **Главные идеи:**
   - [идея 1]
   - [идея 2]

   **Сущности:** [Проект A], [Проект B], [Термин X]

   **См. также:** [ссылка 1], [ссылка 2]
   ```

## Алгоритм для папки/раздела

1. Прочитать `README.md` раздела (там обычно курированный список файлов)
2. Прочитать первые 20 строк каждого файла + его `<!-- summary -->`
3. Сгруппировать файлы по подтемам (если их много)
4. Дать сводку: «Раздел из N файлов покрывает M подтем: …»

## Пример

Пользователь: «кратко расскажи о nautilus»

Действия:
```bash
cat docs/nautilus/README.md  # Read это
ls docs/nautilus/
```

Ответ:
> **TL;DR:** Nautilus Portal Protocol — формальная спецификация федерации репозиториев знаний с native-форматами, которая объединяет 9 working papers (NPP v1.0/1.1, Review Methodology, OKWF, Layer-B и др.).
>
> **Главные идеи:**
> - Федерация, а не слияние: native-форматы сохраняются, унификация только при обращении
> - 23 раздела в формальной спецификации NPP v1.1
> - Companion papers: Cinderella Syndrome (Representative Agent Layer), 21 учитель (Composite Skills)
> - Двойная треугольная архитектура (звезда Давида) для human-AI collaboration
>
> **Сущности:** NPP, OKWF, Cinderella Syndrome, Layer B, Q6-гиперкуб
>
> **См. также:** [`docs/nautilus/npp-v1-1/`](...), [`docs/nautilus/okwf-concept/`](...)
