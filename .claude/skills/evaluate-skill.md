# evaluate-skill

Мета-скилл: оценить, насколько применённый скилл решил задачу пользователя.

## Когда использовать

После выполнения задачи — особенно если применили автоматический скилл-роутер.
Цель — собрать обратную связь для улучшения системы.

## Шаги

1. **Зафиксировать что было запрошено**
   - Точная формулировка пользователя
   - Какой скилл был выбран
   - Какие альтернативы рассматривались

2. **Зафиксировать результат**
   - Что вернул скилл
   - Время выполнения
   - Использованные инструменты (MCP-tools, скрипты)

3. **Оценить по 4 осям**

   | Ось | Шкала | Что значит |
   |---|---|---|
   | **Helpfulness** | 1-5 | Решил ли запрос? |
   | **Honesty** | 1-5 | Достоверны ли факты? |
   | **Completeness** | 1-5 | Покрыто ли всё? |
   | **Efficiency** | 1-5 | Минимально ли усилий? |

4. **Сравнить с альтернативами**

   Если выбран был не оптимальный — записать почему.

5. **Записать в журнал**
   ```jsonl
   {
     "ts": "...",
     "request": "...",
     "skill": "skill-name",
     "scores": {"helpful": 4, "honest": 5, "complete": 3, "efficient": 5},
     "duration_ms": ...,
     "tools_used": [...],
     "feedback": "..."
   }
   ```
   Файл: `.claude/skill_metrics.jsonl`

6. **Если score низкий — предложить улучшение**
   - Скилл не покрывает кейс → создать новый
   - Триггеры неточные → дописать в `trigger_phrases`
   - Шаги неполные → обновить markdown
   - Композиция нужна → создать составной скилл

## Когда оценка должна происходить автоматически

- При первом использовании нового скилла
- Когда пользователь прерывает выполнение
- Когда результат пустой
- Когда время выполнения > 60 секунд

## Когда оценку лучше пропустить

- Тривиальный запрос («покажи статус»)
- Когда пользователь явно не хочет обратной связи
- В составе `daily-routine` — там уже есть встроенная оценка
