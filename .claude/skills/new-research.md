# new-research

Старт нового исследования по теме.

## Когда использовать

- "начни исследование по теме X"
- "нужно изучить Y, с чего начать"
- "хочу разобраться в Z"
- "новая research-note"

## Композиция

1. [`find-gaps`](find-gaps.md) — что уже есть / нет
2. [`summarize`](summarize.md) — что есть, кратко
3. [`generate-rfc`](generate-rfc.md) — если по итогу нужен RFC
4. Создание `research-note.md` через шаблон

## Шаги

1. **Уточнить тему**
   - Конкретно что изучить
   - Зачем (какое решение зависит от итогов)
   - Сроки (одна сессия / неделя / месяц)

2. **Применить `find-gaps`** для темы
   - Что уже описано
   - Чего не хватает
   - Какие связанные темы упомянуты

3. **Если многое уже описано — `summarize`**
   - Прочитать топ-5 файлов
   - Сводное резюме
   - Найти противоречия / неоднозначности → отметить

4. **Если тема новая — пройти по источникам**
   - Внешние URL из `CITATION_INDEX.md` если есть
   - Habr, GitHub: `python scripts/improve_external_compare.py --auto`

5. **Создать research-note**
   ```bash
   python scripts/improve_template_init.py --type research-note \
     --slug docs/research/<slug>.md \
     --vars title="<тема>" status=draft
   ```

6. **Структурировать заметку**
   - **Контекст:** зачем изучаем
   - **Что узнали:** ключевые находки
   - **Источники:** все консультированные
   - **Открытые вопросы:** что не понятно
   - **Следующие шаги:** план

7. **Если тема созрела до RFC** — применить `generate-rfc`

8. **Связать с корпусом**
   - Обновить `docs/INDEX.md` — добавить ссылку
   - Связать с релевантным разделом
   - Если новый термин — `glossary-entry`

## Шаблон заметки

```markdown
---
template: research-note
title: "..."
status: draft
---

# [Тема]

## Контекст
...

## Уже известно
- [Тезис 1] — [`docs/...md`]

## Новые находки
1. **[Находка 1]** — [источник]
2. **[Находка 2]**

## Противоречия / неоднозначности
- [Вопрос]

## Открытые вопросы
- ?

## Источники
- [Внешний 1](url)
- [Внутренний] — [`docs/...md`]

## Следующие шаги
- [ ] ...
```
