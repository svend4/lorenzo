# analyze-project

Анализирует конкретный проект из базы знаний Lorenzo.
Когда использовать: пользователь просит проанализировать, оценить, объяснить
проект (Yodoca, NGT, AgentFS, Svyazi, и т.п.) или файл из docs/.

## Шаги

1. **Найти файл проекта**
   ```bash
   find docs/ -name "*.md" | xargs grep -l "<PROJECT_NAME>" 2>/dev/null | head -5
   ```

2. **Прочитать файл** — используй Read tool для основного файла проекта

3. **Собрать контекст из аналитики**
   - Прочитать строки с проектом из `docs/ENTITIES.md` (упоминания)
   - Прочитать строки с проектом из `docs/CONTACTS.md` (автор, слой)
   - Найти проект в `docs/SIMILAR.md` (похожие)
   - Найти проект в `docs/DECISIONS.md` (решения)
   - Проверить `docs/contacts/<slug>.md` если есть

4. **Запустить метрики на файле** (опционально)
   ```bash
   python scripts/improve_metrics.py
   # затем прочитать нужную строку из docs/METRICS.md
   ```

5. **Синтезировать ответ** — объединить данные из шагов 2-4:
   - Что делает проект (из файла)
   - Технические особенности
   - Место в архитектуре Svyazi (слой, интеграции)
   - Похожие проекты и чем отличается
   - Контакт автора и статус связи
   - Рекомендация: стоит ли контактировать / использовать

6. **Предложить следующий шаг**
   - Если контакт не установлен → показать черновик из `docs/contacts/`
   - Если файл неполный → предложить запустить `improve_llm_enrich.py --section`

## Формат ответа

```
## {Название проекта}

**Слой:** {memory | rag | orchestration | ...}
**Автор:** @{handle}
**Упоминаний:** {N} файлов

### Суть
{2-3 предложения}

### Ключевые факты
- {факт 1}
- {факт 2}

### Место в Svyazi
{как встраивается}

### Похожие проекты
- {проект} — {чем отличается}

### Статус контакта
{не писали / написали / ответили}
→ [открыть черновик](docs/contacts/{slug}.md)
```
